# Security Policy

## Supported Versions

Security fixes target the latest published version.

## Reporting A Vulnerability

Do not report suspected vulnerabilities in public issues.

Use GitHub Security Advisories for this repository when available. If advisories are not enabled, contact the package maintainer privately through the npm owner profile or another private maintainer channel listed for the project.

Please include:

- affected version or commit
- impact and attack scenario
- reproduction steps
- whether tokens, page content, workspace IDs, or personal data may have been exposed

## Secret Handling

Never include Notion API tokens, private page IDs, workspace IDs, audit logs with sensitive content, or `.env*` files in reports unless a maintainer explicitly asks for a redacted sample through a private channel.
